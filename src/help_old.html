Равнодоступная адресная машина(РАМ)

Random access memory machine (RAM)

Структура РАМ

Входной поток: x[0],x[1],x[2],... (входные данные)
Указатель входного потока: i
Начальное значение указателя входного потока: 0
Текущее значение входного потока (входное значение): x[i]
Выходной поток: y[0],y[1],y[2],... (ячейки для выходных данных)
Указатель выходного потока: j
Начальное значение указателя выходного потока: 0
Текущее значение выходного потока (выводимое значение): y[j]
Оперативная память: ...,r[-2],r[-1],r[0],r[1],r[2],... (регистры)
Аккумулятор (сумматор): r[0]
Программа: 1: I[1];
           2: I[2];
              ...
              (команды)
Начальное содержимое памяти: неинициализирована
Структура команды: [метка:] код_операции модификатор операнд [// комментарии]

Коды операций

Коды основных операций
LOAD - загрузка сумматора(нулевого регистра),
STORE - запоминание сумматора(нулевого регистра),
ADD - сложение,
SUB - вычитание,
JUMP - переход,
JZERO - переход по нулю,
JGTZ - переход по положительному значению,
READ - чтение,
WRITE - запись,
MULT - умножение,
DIV - деление,
HALT - остановка.
Коды дополнительных операций
MOD - остаток от деления.
Модификаторы: =, пробел, *
Операнды: числовая константа, имя метки , код символа('s')
Тип данных: целые числа

Система команд РАМ

Команда Аргумент Комментарий
Работа с памятью
LOAD =a r[0]:=a
LOAD a r[0]:=r[a]
LOAD *a r[0]:=r[r[a]]
STORE a r[a]:=r[0]
STORE *a r[r[a]]:=r[0]
Сложение и вычитание
ADD =a r[0]:=r[0]+a
ADD a r[0]:=r[0]+r[a]
ADD *a r[0]:=r[0]+r[r[a]]
SUB =a r[0]:=r[0]-a
SUB a r[0]:=r[0]-r[a]
SUB *a r[0]:=r[0]-r[r[a]]
Умножение и деление
MULT =a r[0]:=r[0]*a
MULT a r[0]:=r[0]*r[a]
MULT *a r[0]:=r[0]*r[r[a]]
DIV =a r[0]:=r[0]/a
DIV a r[0]:=r[0]/r[a]
DIV *a r[0]:=r[0]/r[r[a]]
MOD =a r[0]:=r[0]%a
MOD a r[0]:=r[0]%r[a]
MOD *a r[0]:=r[0]%r[r[a]]
Примечание: x / y - целая часть от деления x на y
x % y - остаток от деления x на y
Деление на ноль вызывает ошибку!
Прыжки
JUMP m перейти на метку m
JZERO m если r[0]=0 перейти на метку m, иначе - к следующей комманде
JGTZ m если r[0] > 0 перейти на метку m, иначе - к следующей комманде
Чтение и запись лент
READ a r[a]:=x[i]; i:=i+1
READ *a r[r[a]]:=x[i]; i:=i+1
WRITE =a y[j]:=a; j:=j+1
WRITE a y[j]:=r[a]; j:=j+1
WRITE *a y[j]:=r[r[a]]; j:=j+1
Завершение программы
HALT остановка машины RAM

Формат входной ленты 

На входной ленте хранятся целые числа
Задавать их можно тремя способами:
1) Числа (например: 143, -567 или 42)
2) Коды символов(например: 'q' или '3')
3) Строки из кодов символов(например: "123" или "abcde")

Между данными на входной ленте ставятся пробелы или переносы строк

Макросы

Вообще говоря, в машине RAM нет макросов... Как следствие, в рамках учебного процесса их использование нежелательно
Тем не менее, в данной программе они реализованы.
Используются они с помощью команд "#macro macroName" и "#endmacro", а вызываются как обычные команды, с помощью написания macroName в коде программы
В этом случае в программу вставляется весь код макроса.
При этом всем меткам тела макроса присваиваются новые названия, чтобы один и тот же макрос можно было вызывать несколько раз
Авторы программы решили, что делать вместо эмулятора машины РАМ С-подобный компилятор - нецелесообразно, поэтому не стали делать вложение макросов, параметры для макросов, функции, нормальный препроцессор, сборщик мусора и прочие радости жизни, к которым привыкли программисты... Да, ещё они пренебрегли некорректным с точки зрения пользователей отображением строк ошибок в некоторых случаях некорректного использования макросов
