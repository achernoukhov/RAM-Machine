Равнодоступная адресная машина (РАМ)

Random access memory machine (RAM)

Структура РАМ

Выходная лента
Элемент Обозначение Описание
Входная лента x[1],x[2],...,x[n] Содержит входные данные. Данные представлены последовательностью чисел и/или символов
y[1],y[2],...,y[n],... Содержит выходные данные. Данные представлены последовательностью чисел и символов
Регистры ...r[-n],...,r[-1],r[0],r[1],...,r[n]... Регистры представляют память машины. Машина сохраняет временные данные в регистрах во время выполнения алгоритма. Изначально вся память неинициализирована. Нулевой регистр является сумматором.
Команды Нет Команды представляют алгоритм.

Структура команд

[метка:] имя_команды аргумент1 аргумент2 ... аргументN [//комментарии]

Описание:
"[..]" Необязательный элемент
метка Любое имя. Метка указывает на определённую строку в коде. С помощью меток можно "именовать" строки.
имя_команды Любая допустимая команда (см. ниже)
аргументN N-й аргумент команжы.
комментарии Любые комментарии. все символы в строке после двойной косой черты (//) интерпретируются как комментарии. Комментарии игнорируются машиной и нужны только для программиста
Пример:
reading: read *0 // чтение с входной ленты
Здесь "reading" - имя метки, - "read" - команда, - "*0" - аргумент, - а всё оставшееся - комментарии

Замечание: Все допустимые команды имеют один или ни одного аргумента. Поэтому "N-й аргумент" - это лишь обобщение на случай включения в РАМ некоторых команд с 2 или более аргументами

Команды

load загрузка в нулевой регистр (сумматор)
store сохранение содержимого сумматора в другой регистр
add сложение
sub вычитание
mult умножение
div деление
mod остаток от деления
jump переход (прыжок) на указанную строку в коде
jzero прыжок, если сумматор в данный момент содержит 0
jgtz прыжок, если сумматор в данный момент содержит положительное число
read считывание со входной ленты
write вывод числа на выходную ленту
writesym вывод символа на выходную ленту
halt окончание программы

Машина РАМ выполняет команды последовательно строка за строкой, пока не встретит halt - тогда программа завершается, - или jump, jzero или jgtz - тогда переходит на указанную строку.

jump m goto m Переход на строку кода, определяемую меткой m
jzero m if r[0]=0 goto m else goto next Если сумматор содержит 0, тогда переход на строку кода, определяемую меткой m, иначе - переход на следующую строку кода

Входная лента

Входная лента РАМ представляет собой последовательность чисел. Но в данном приложении возможно включать во входную ленту также символы и даже строки (совокупности символов). Сначала РАМ конвертирует все входные данные в последовательность чисел, а затем работает непосредственно с этой последовательностью.
Чтобы задать входную ленту, нужно ввести набор выражений, разделённых любым количеством пробелов, табуляций или символов конца строки.
Каждое из этих выражений должно иметь одну из описанных ниже форм:
Форма Примеры
Число 1, 3, 155 и т.д.
символ, заключённый в апострофы (') 'а', '7', '&' и т.д.
Строка (совокупность символов), заключённая в кавычки (") "Hello!", "abc123!@#", "2+2=4" и т.д.
